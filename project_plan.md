The relationships between the tables are as follows:

* A user can have many rides (one-to-many).
* A driver can have many rides (one-to-many).
* A ride is associated with one user and one driver (many-to-one).
* A payment is associated with one ride (many-to-one).
* A rating is associated with one ride (many-to-one).

### Auth APIs
1. **POST /auth/register** - Create a new user account using email, phone number, or social media profiles.
2. **POST /auth/login** - Login to an existing user account.
3. **POST /auth/forgot-password** - Send a password reset link to the user's email.
4. **POST /auth/reset-password** - Reset the user's password.

### User APIs
5. **GET /users/me** - Get the currently logged-in user's profile information.
6. **PUT /users/me** - Update the currently logged-in user's profile information.
7. **GET /users/me/bookings** - Get a list of the user's previous bookings.
8. **POST /users/me/promotions** - Get a list of available promotions and discounts for the user.

### Driver APIs
9. **GET /drivers/nearby** - Get a list of available drivers near the user's location.
10. **GET /drivers/{driver_id}** - Get a specific driver's profile information.

### Ride APIs
11. **POST /rides** - Book a new ride by entering pickup and drop-off locations.
12. **GET /rides/{ride_id}** - Get the details of a specific ride.
13. **PUT /rides/{ride_id}/cancel** - Cancel a booked ride.
14. **GET /rides/{ride_id}/track** - Get the real-time location of the driver assigned to the ride.

### Payment APIs
15. **POST /payments** - Make a payment for a ride using a credit card or other payment method.
16. **GET /payments/{payment_id}** - Get the details of a specific payment.
17. **POST /payments/{payment_id}/refund** - Request a refund for a payment.
18. **GET /payments/methods** - Get a list of available payment methods.
19. **POST /payments/methods** - Add a new payment method (e.g. credit card) to the user's account.
20. **GET /rides/{ride_id}/receipt** - Get the receipt for a completed ride.

These APIs cover the main functionality of the ride booking app, including user registration, vehicle selection, ride booking, driver assignment, real-time tracking, payment integration, ride cancellation, and rating system. The APIs are designed to be RESTful and follow standard naming conventions. The API endpoints are grouped by category (auth, user, driver, ride, payment) for clarity and organization.
